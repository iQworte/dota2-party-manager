# Dota2 Party Manager

Desktop-приложение для Windows: ведёт статистику побед и поражений с **напарниками из пати** в **ranked**-матчах Dota 2.

## Скачать

Готовая portable-версия для Windows: [v1.3.0](https://github.com/iQworte/dota2-party-manager/releases/latest)

## Сборка (portable)

Готовый файл после сборки:

```
dist/Dota2PartyManager-Portable-1.3.0.exe
```

Сборка из исходников:

```bash
npm install
npm run build:win
```

Portable-версия не требует установки: достаточно скопировать `.exe` в любую папку и запустить.

## Первый запуск

1. Запустите `Dota2PartyManager-Portable-1.3.0.exe`.
2. Нажмите **«Найти Dota»** или укажите путь к папке Dota 2 вручную  
   (обычно `...\steamapps\common\dota 2 beta`).
3. Нажмите **«Установить GSI»** — приложение создаст cfg-файл интеграции в  
   `game/dota/cfg/gamestate_integration/`.
4. **Перезапустите Dota 2**, если клиент уже был открыт.
5. Зайдите в игру (достаточно одного матча или лобби) — после этого в шапке приложения появится статус **«GSI подключён»**.

> **Совет:** если GSI не активируется, добавьте в Steam параметры запуска Dota 2: `-gamestateintegration`

## Как работает статистика

После каждого **ranked**-матча, сыгранного **в пати** (2+ человека в группе):

1. Dota через GSI сообщает приложению о завершении игры.
2. Матч попадает в **очередь обработки**.
3. Приложение запрашивает данные матча у [STRATZ API](https://stratz.com/api) и определяет напарников по `partyId`.
4. Каждому напарнику начисляется победа или поражение.

**Не учитывается:**

- обычный (нерейтинговый) All Pick, Turbo и другие режимы;
- solo-очередь (игра без пати);
- матчи, где STRATZ ещё не проиндексировал данные (приложение повторяет запрос автоматически).

Очередь **сохраняется** между перезапусками приложения — матчи не теряются, если вы закрыли программу до завершения обработки.

## Интерфейс

| Элемент | Описание |
|--------|----------|
| **GSI подключён / не подключён** | Связь с клиентом Dota 2 |
| **Настройки (шестерёнка)** | STRATZ API token и прокси |
| **Журнал действий** | Важные события: STRATZ, очередь, импорт, GSI |
| **Текущий аккаунт** | Dota ID и ник из GSI |
| **Последний матч** | ID последнего обработанного матча |
| **Очередь обработки** | Сколько матчей ждут ответа STRATZ |
| **Таблица игроков** | Ник, Dota ID, W/L, winrate, дата последнего матча |

### Winrate и MMR

Winrate считается как `победы / (победы + поражения)`.  
Рядом показывается условный MMR-показатель: `(победы − поражения) × 25`.

### Поиск

Поле **«Поиск по нику или ID»** фильтрует таблицу в реальном времени.

### Редактирование и удаление

- **Карандаш** — изменить отображаемый ник, победы и поражения вручную.
- **Корзина** — удалить игрока из статистики.

### Настройки STRATZ и прокси

Кнопка **шестерёнки** в шапке слева открывает настройки:

- **STRATZ API token** — обязателен, бесплатно на [stratz.com/api](https://stratz.com/api);
- **Прокси** — опционально (IP, порт, логин, пароль).

### Журнал действий

Кнопка **журнала** в шапке показывает важные события: запросы STRATZ, очередь матчей, импорт, подключение GSI и сброс статистики.

### Импорт последних матчей

Кнопка **«Импорт последних матчей»**:

- загружает последние N (до 500) **ranked**-матчей указанного Dota ID из STRATZ;
- при подключённом GSI Dota ID подставляется автоматически;
- пропускает нерейтинговые и уже учтённые матчи;
- добавляет остальные в очередь обработки;
- показывает прогресс загрузки `Загрузка (i/n)` при получении истории из STRATZ.

> Для импорта нужен STRATZ API token и опция **Expose Public Match Data** в Dota 2 (Настройки → Социальное).

### Сброс статистики

**«Сбросить всю статистику»** удаляет всех игроков, историю и очередь.  
Требуется подтверждение: галочка + ввод слова `СБРОС`.

## Где хранятся данные

| Режим | Путь |
|-------|------|
| Portable (`.exe`) | папка `data/` **рядом с exe** |
| Разработка (`npm start`) | папка `data/` в корне проекта |

Файлы:

- `data/stats.json` — статистика игроков, обработанные и ожидающие матчи;
- `data/config.json` — путь к Dota 2, настройки GSI, STRATZ token и прокси.

Для переноса на другой ПК скопируйте `.exe` вместе с папкой `data/`.

## Разработка

```bash
npm install
npm start              # Electron-приложение
npm run dev:server     # только HTTP-сервер (порт 38456)
npm run start:dev      # Electron + DevTools (F12)
npm run check          # проверка синтаксиса
```
